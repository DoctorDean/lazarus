The pipeline — composed from revived bricks

Because every revival emits the same contract, each is a composable brick. One YAML wires them; one command runs them on the right host (here: ScanNet + dMaSIF on GPU + fpocket, then a consensus adapter) — an in-silico binder triage from methods that were individually unrunnable a week ago.

structure
4ZQK chain A
→
ScanNet — interface
dMaSIF — interface (GPU)
fpocket — druggable pockets
consensus — fuse & triage
Live on PD-L1, the pipeline concluded: 27 interface residues clearly localized (115, 123, 56, 121, 113…), but 0 druggable pockets among them — “a flat protein-protein interface: an antibody / biologic target, not a small-molecule one.” That's textbook immuno-oncology (PD-1/PD-L1 is an antibody target), reproduced from dead code.
04

Give back

The fixes Lazarus found are real and wanted. For the genuinely-abandoned repos, it prepares maintainer-ready PRs — the fix plus a CI smoke test, so the method can't silently rot again. (dMaSIF is skipped — no-derivatives license; fpocket's upstream is alive.)

MaSIF · issue #85
The rotted PDB download (Biopython's retired wwPDB path) replaced with a direct RCSB fetch — verified to revive the built-in flow at ROC-AUC 0.9137 — plus a CI smoke test asserting ≥ 0.8.
ScanNet · issue #15
library_folder='' made to auto-detect the repo root, fixing single-PDB inference from any directory — verified — plus a CI smoke test.

06

How Lazarus works

Sandbox
Disposable container (CPU or GPU); expensive successes snapshotted so a failure never re-pays the build.
Commit-era pinner
Reconstructs the dependency universe as of the repo's last commit — the reasoning that beat the cu111/KeOps/cppyy tangle.
Repair loop
build → run → read traceback → patch → retry, bounded, isolated to the container.
Capability locator
Finds where "input → the famous output" happens and carves the minimal path.
Contract emitter + Compose
Uniform contracts (CPU/GPU) become bricks a YAML wires into pipelines.

Runs on a laptop; executes anywhere — local, a lab box, a cloud VM, or a GPU rental — over one --docker-host flag. All four were driven from a MacBook and executed on a native-x86 + GPU workstation.
