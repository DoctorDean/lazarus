01

The patient

Method

  • CapabilityMaSIF-site — per-vertex protein interaction-site prediction from molecular surface fingerprints
  • PaperGainza et al., Nature Methods 2020
  • UseTriage step in a binder-discovery pipeline
  • WeightsPretrained all_feat_3l, shipped in-repo

Why it wouldn't run

Published, open, and effectively unrunnable — a stack pinned to an era that modern machines and package managers no longer resolve:

Python 3.6TensorFlow 1.12MSMSAPBSPDB2PQRreducePyMesh (from source)slurm data-prep

The real capability is buried in shell scripts, invoked by hand — never exposed as an API.

02

The procedure — 18 autonomous turns

Lazarus ran a build → execute → read-traceback → repair loop in a disposable container, using only its own container tools. No human input.

  • Orient

    Located the real capability

    Read data_prepare_one.sh, predict_site.sh, color_site.sh and traced the path from a PDB to per-vertex scores.

  • Inventory

    Probed the environment

    Confirmed msms / reduce / apbs / pdb2pqr / multivalue on PATH, TensorFlow 1.12 imports (host has AVX), pretrained weights present.

  • Diagnose

    One broken link: the PDB download

    The built-in fetch points at a rotted RCSB URL (issue #85) — the single thing standing between the repo and a result.

  • Repair

    Routed around it

    Fetched 4ZQK from the current RCSB endpoint and fed it through the script's --file mechanism — also the right API design.

  • Prepare

    Built the surface + features

    reduce → MSMS → APBS → triangulate → precompute produced all feature arrays incl. ground-truth iface_labels. ▸ snapshot: site-dataprep-ok

  • Predict

    Ran the pretrained network

    predict_site.sh → 2,336 per-vertex interaction-site scores.

  • Prove

    ROC-AUC = 0.9137 on 4ZQK_A

    Above the 0.80 bar and the paper's ~0.85. ▸ snapshot: site-ready

  • Deliver

    Emitted the integration contract

    A self-contained, callable package written back to the operator's machine.

04

How Lazarus works

Sandbox
Disposable container; every expensive success is snapshotted so a later failure never re-pays the build.
Commit-era pinner
Reconstructs the dependency universe as it was on the repo's last commit, from the PyPI timeline.
Repair loop
Build → run → read traceback → patch → retry, bounded, isolated to the container.
Capability locator
Finds where "input → the famous output" happens and carves the minimal path to it.
Contract emitter
Module + CLI + pinned container + smoke test encoding the user's sanity check.

Runs on a laptop; executes anywhere. This resurrection was driven from a MacBook and executed on a native-x86 workstation over a single --docker-host flag — because MaSIF's binaries (MSMS, TF) need native x86 that Apple-silicon emulation can't provide. Lazarus's sandbox is host-pluggable, so the method runs wherever the method can run.
