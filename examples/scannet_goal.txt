Resurrect ScanNet's protein-protein binding-site prediction (jertubiana/ScanNet) so it
runs on a single PDB chain and produces per-residue binding-site probabilities.

The repo lives inside this container (locate it, e.g. find the file predict_bindingsites.py).
The entry point is predict_bindingsites.py. Use the protein-protein interface model in
STRUCTURE-ONLY mode: the default `--mode interface` with `--noMSA` (no MSA / HHblits).
Pretrained weights are shipped in the repo under models/ (e.g. ScanNet_PPI_noMSA) — no
download or training needed.

Target: 4ZQK chain A (PD-L1).

Known issues to expect and work around (verify by reading the code / tracebacks):
- The built-in structure auto-download is broken (returns "structure doesn't exist" then
  crashes on a None path). Do NOT rely on it. Fetch the structure yourself from the current
  RCSB endpoint (https://files.rcsb.org/download/4ZQK.pdb) and pass a LOCAL file; ScanNet's
  chain syntax is like `/path/to/4zqk.pdb_A`.
- Even with --noMSA the code may still touch MSA_folder paths configured in
  utilities/paths.py. If it errors on MSA or missing paths, set/patch utilities/paths.py
  (structures_folder, predictions_folder, MSA_folder, etc.) so inference runs.
- Legacy stack (Python 3.6 / TF 1.14 / Keras 2.2.5). Expect a flood of harmless numpy
  FutureWarnings — ignore them.

ScanNet's per-residue probabilities against those interface labels. You must reach
