How it works

A dead repo → a callable tool, from a link

🔗
Point it at a GitHub URL
github.com/gcorso/DiffDock — open, cited, and 2 years too stale to run.
📄
Read the repo & the paper
The Scout writes its own goal — and a falsifiable success test it must pass.
📌
Pin dependencies to the repo's era
Reconstruct the exact stack from the commit date — torch 1.13 · CUDA 11.7, not today's.
📦
Spin up a disposable sandbox
A clean container — CPU or GPU — isolated from everything else.
🔁
Build → run → read the error → fix → retry
A repair loop that patches real bugs, not just installs — until it actually runs.
🎯
Carve the capability
Find where input → the famous output happens, and keep the minimal path.
✅
Emit a contract
Pinned container + clean API + the self-test — so every revived tool composes.
🏅
Reproduce the paper & give back
Match the published numbers, then send the fix upstream as a pull request.
